# 実行手順

## 0. モード判定

- 引数 `daily` または「今日のコスト確認」系の依頼 → **手順 1 の日次コマンドと手順 4 の表示のみ**（分析はしない）
- 引数 `monthly` または「今月のコスト確認」系の依頼 → **手順 1 の月次コマンドと手順 4 の表示のみ**
- 引数なしで「分析・削減提案」の依頼 → 全手順を実行

日次のみの表示には: 今日のコスト・直近7日平均・高コスト Top3 を添える。
月次のみの表示には: 今月のコスト・前月比・月末予測を添える。

## 1. データ収集

日次と月次の両方のレポートを取得:

```bash
# 日次レポート
npx ccusage@latest

# 月次レポート
npx ccusage@latest monthly
```

## 2. コスト分析

以下の観点で分析:

### 2.1 高コストセッションの特定
- $20以上の日をリストアップ
- 何が原因だったか推測（大量のファイル操作、長時間セッションなど）

### 2.2 キャッシュ効率の分析
- Cache Read / Total Tokens の比率を計算
- キャッシュヒット率が低い場合は改善の余地あり

### 2.3 モデル使用比率
- haiku-4-5 と sonnet-4-5 の使用比率
- sonnet が多い場合、haikuで十分なタスクがないか検討

### 2.4 時系列トレンド
- 週ごとのコスト推移
- 増加傾向にある場合は注意喚起

## 3. 削減提案

分析結果に基づき、以下のような提案を提示:

### 効率化の提案
- **高コスト日の削減**: 特定の日のコストが突出している場合、その原因と対策
- **モデル選択の最適化**: 簡単なタスクには haiku を使用
- **セッション分割**: 長時間セッションを避け、適度に区切る
- **キャッシュ活用**: プロンプトキャッシュを意識した使い方

### 具体的なアクション
1. 高コストな操作パターンの見直し
2. Task tool の積極的活用（サブエージェントで効率化）
3. 不要なファイル読み込みの削減
