# pr-develop — 開発セッション開始

Issueに記録した「意図」を現在のセッションに注入するスキル。「やること」と「やらないこと」を明示することでスコープ外の実装への誘惑を抑止する。

## 引数

`$ARGUMENTS` からIssue番号を読み取る（`#42` または `42` の形式）。
番号がなければ `gh issue list --assignee @me --state open` を実行し、ユーザーに選んでもらう。

## Step 1: Issueの内容を取得

```bash
gh issue view <番号>
```

## Step 2: スコープを要約して表示

以下の形式でセッションのスコープを宣言する:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  開発セッション開始: Issue #<番号>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

課題:
  <1行>

解決の意図:
  <1行>

やること:
  - <項目>
  - <項目>

やらないこと（この実装では触れない）:
  - <項目>

完了条件:
  - <条件>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 3: feature ブランチの作成

現在のブランチを確認する:

```bash
git branch --show-current
```

main / master / develop など base ブランチにいる場合は feature ブランチを作成する:

```bash
git checkout -b feature/#<番号>-<issue-title-kebab-case>
```

すでに適切なブランチにいる場合はスキップし、その旨を伝える。

## Step 4: セッション開始を宣言

以下を伝えてセッションを開始する:

> スコープを確認しました。実装を開始してください。
>
> リマインダー:
> - コミットは意図単位で。準備ができたら `/commit` を使ってください
> - スコープ外の作業が必要になったら実装を止め、新しいIssueを `/pr-plan` で作成してください
> - PRを作る準備ができたら `/pr-create` を使ってください
