# Session Analyzer

全 Claude セッションのユーザーメッセージを抽出し、パターンを分析して最適化提案を行う。

## 実行手順

### 1. セッションデータ抽出

```bash
npx tsx "$HOME/.claude/skills/session-analyzer/scripts/extract-sessions.ts" > tmp/sessions-output.txt 2>&1
```

出力が大きいため `tmp/sessions-output.txt` に保存してから読む。

### 2. パターン分類

抽出したメッセージを以下の4カテゴリに分類する:

#### スキル (Skills) に向いているもの
- **繰り返し実行されるワークフロー**（週次・日次のルーティン）
- **定型の手順**があるもの（毎回同じステップを踏む）
- **スラッシュコマンドで呼び出す**明確なトリガーがあるもの
- 例: worklog 更新、1on1 準備、commit & push

#### MCP プラグイン に向いているもの
- **外部サービスとの永続的な接続**が必要なもの
- **認証・セッション管理**が必要なもの
- **双方向のリアルタイムデータ**が必要なもの
- 例: Notion, Slack, GitHub, Google Calendar への頻繁なアクセス

#### エージェント (Agents) に向いているもの
- **複数ステップの並列実行**が必要なもの
- **長時間・高コストのタスク**で main session を圧迫するもの
- **独立した文脈**で動ける専門タスク
- 例: コードレビュー, モジュール移行, PR 修正

#### CLAUDE.md に入れるもの
- **すべてのセッションで常に適用したいルール**
- **ツール選択の優先順位**（jq 優先、git -C 禁止など）
- **プロジェクト構造・命名規約**
- **繰り返し修正を求めるパターン**（Claude が同じ間違いを繰り返すもの）

### 3. レポート出力

分析結果を以下の形式でまとめる:

```markdown
## セッション分析レポート

### 全体統計
- 期間: YYYY-MM-DD 〜 YYYY-MM-DD
- セッション数: N
- 主なプロジェクト: ...

### カテゴリ別分類

#### ✅ 既にスキル化されている
（現在のスキルと実際の使用状況を照合）

#### 🆕 スキル化を推奨
| パターン | 頻度 | 理由 |
|----------|------|------|

#### 🔌 MCP プラグイン化を推奨
| サービス | 用途 | 現在の方法 |
|----------|------|------------|

#### 🤖 エージェント化を推奨
| タスク | 並列化メリット | 推定コスト削減 |
|--------|--------------|----------------|

#### 📝 CLAUDE.md に追加を推奨
| ルール | 根拠（何度修正を求めたか） |
|--------|--------------------------|
```

### 4. 改善提案生成

分類結果をもとに、具体的なアクションを提案する。

**優先度の判断基準:**
- **高**: 週3回以上繰り返している / Claude が同じ間違いを3回以上している
- **中**: 週1〜2回 / 毎回手順を思い出すコストがかかる
- **低**: 月1〜数回 / あると便利だが緊急ではない

各提案に以下を付与する:

| 提案 | カテゴリ | 優先度 | 実装難易度 | 推奨アクション |
|------|---------|--------|-----------|----------------|
| （観察されたパターン） | スキル/MCP/エージェント/CLAUDE.md | 高/中/低 | 低/中/高 | PR直接作成 / issue起票 |

**推奨アクションの基準:**

| 種別 | 推奨アクション |
|------|--------------|
| CLAUDE.md への追記（1行程度） | PR を直接作成 |
| スキル説明文の改善 | PR を直接作成 |
| 新スキル・新エージェント提案 | GitHub issue を起票 |
| 大きなアーキテクチャ変更 | GitHub issue を起票 |

### 5. 優先度トップ3

```markdown
### 優先度トップ3
1. 最も ROI が高い改善（推奨アクション: ...）
2. 次点（推奨アクション: ...）
3. 3位（推奨アクション: ...）
```

## 注意

- セッションファイルにはプライベートな情報（コード・会議内容等）が含まれる
- 分析結果はローカルのみで処理し、外部送信しない
- 大量のセッションがある場合は直近 90 日に絞っても良い
